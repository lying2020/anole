# Anole 推理脚本与算法完整梳理

本文档说明 `inference.py` 的用途、调用链，以及从「输入一个 case」到「得到结果」的全流程，包括 token 变化、剪枝与数量关系。

---

## 一、推理脚本是干什么的

`inference.py` 是一个**多模态（图文）推理入口**：

- **输入**：一个 JSON 文件，描述多模态 prompt（可交替包含多段「文本」和「图像」）。
- **输出**：
  - 模型自回归生成的**文本**直接打印到 stdout；
  - 模型生成的**图像**以「BOI + 1024 个 image token + EOI」形式解码为 PNG，保存到 `--save_dir`，并打印其路径。

因此，它做的是：**给定图文混合的 prompt，让 Chameleon/Anole 生成一段可能包含文字和图片的回复**（例如：先分析图中内容，再生成一张新图，再打分说明）。

---

## 二、整体流程概览

```
输入 JSON
    ↓
解析 → batch_prompt_ui（文本 + 图像 file: 路径）
    ↓
tokens_from_ui() → 转成统一 token 序列（含 BOS、文本 token、BOI+图像 token+EOI）
    ↓
ChameleonInferenceModel.stream(batch_prompt_ui, options)
    ↓
内部：Generator → 动态在 TextDecoder / ImageDecoder 间切换
    ↓
TextDecoder：ChameleonGenerator 自回归生成，直到 EOS 或 BOI
    ↓
若生成 BOI → 切到 ImageDecoder
    ↓
ImageDecoder：固定生成 1024 个 image token，再强制 EOI，再回到 TextDecoder
    ↓
stream() 逐 token yield 给 inference.py
    ↓
inference.py 收集为 tokens，按 BOI/EOI 做 split_token_sequence
    ↓
按段 decode_text / decode_image → 打印文本、保存图片
```

下面按「入口 → 模型 → token 与剪枝」拆开说。

---

## 三、入口与配置（inference.py）

### 3.1 命令行与输入格式

- `-i/--input`：多模态输入的 JSON 文件路径（必需）。
- `-s/--save_dir`：生成图片保存目录，默认 `./outputs/inference/`。

JSON 格式示例（`inputs/input.json`）：

```json
[
  { "type": "text", "content": "你的文本指令，例如：分析这张图并生成一张新图..." },
  { "type": "image", "content": "assets/stop_sign.png" }
]
```

支持多段 `text` 与 `image` 交替；`image` 的 `content` 为本地路径，在代码里会转为 `file:绝对路径` 再交给 chameleon。

### 3.2 模型与 tokenizer 路径（constants.py）

- `MODEL_7B_PATH`：7B 权重目录（如 `params.json`、`consolidate_params.json`、`consolidated.pth`）。
- `TOKENIZER_TEXT_PATH`：文本 tokenizer（如 `text_tokenizer.json`，HF Tokenizers 格式）。
- `TOKENIZER_IMAGE_CFG_PATH` / `TOKENIZER_IMAGE_PATH`：VQGAN 配置与权重（`vqgan.yaml`、`vqgan.ckpt`），用于图像 ↔ 1024 个 image token 的编解码。

## 四、Chameleon 推理库调用链（追根溯源）

### 4.1 ChameleonInferenceModel（chameleon/inference/chameleon.py）

- **作用**：封装模型加载、TokenManager（文本+图像 tokenizer）、以及可选的多进程/多线程 worker。对外提供 `stream()`、`generate()`、`decode_text()`、`decode_image()`。
- **初始化**：
  - 用 `loader.load_model(path)` 加载 `Transformer`（见 4.5）；
  - 用 `TokenManager(tokenizer_path, vqgan_cfg_path, vqgan_ckpt_path)` 管理文本 BPE 与图像 VQGAN；
  - `self.vocab` 来自 TokenManager，类型为 `VocabInfo`（见 4.2）。

### 4.2 VocabInfo / VocabTranslation（chameleon/inference/vocab.py）

- **VocabInfo**：从 `text_tokenizer.json` 的 `model.vocab` 解析出的词表信息。
  - 特殊 token：`<s>`(BOS)、`</s>`(EOS)、`<racm3:break>`(BOI)、`<eoss>`(EOI)、`<pad>` 等。
  - `image_tokens`：词表中以 `"IMGIMG"` 开头的 token，对应 VQGAN codebook 的 8192 个码本 id 映射到 BPE id（约 4–8195）。
  - `text_tokens`：`all_tokens - image_tokens - special_tokens`，即只允许生成「文本」时的候选集。
- **VocabTranslation**：在 VQGAN codebook id（0–8191）与 BPE 词表 id（如 4–8195）之间做双向映射，供 `tokenize_image` / `decode_image` 时使用。

### 4.3 TokenManager：从 UI 到 token 序列（chameleon/inference/chameleon.py）

- **tokens_from_ui(inputs)**：
  - 先加 BOS；
  - 对每个 segment：
    - `type=="text"` → `tokenize_text(value)` → 文本 BPE ids；
    - `type=="image"` → 若为 `file:...` 则打开 PIL 图，再 `tokenize_image(pil)`。
  - **tokenize_image(pil)** 返回：`[BOI] + convert_img2bpe(vqgan_encode(pil)) + [EOI]`。  
    VQGAN 把一张图编码成 **32×32=1024** 个 codebook id，再通过 VocabTranslation 转成 BPE id，所以一段图像在序列里是 **1 + 1024 + 1 = 1026 个 token**。

因此：

- **输入侧**：每个 image 在序列中占 **1026 token**（BOI + 1024 + EOI）。
- **输出侧**：每张生成的图同样是 BOI + 1024 个 image token + EOI，解码时只取 1024 那段做 VQGAN 解码。

### 4.4 stream() 与 Generator（chameleon/inference/chameleon.py）

- **stream(batch_prompt_ui=..., options=...)**：
  - 把 `batch_prompt_ui` 转成 `batch_input_ids`：对每个 sample 调 `token_manager.tokens_from_ui(prompt_ui)`，得到 list of list of int。
  - 把 (options, batch_input_ids) 发给 worker；worker 内用 **Generator** 迭代。
  - 主进程从队列取 token，逐个 `yield` 给调用方（inference.py 就在这里收集）。

- **Generator**：
  - 根据 `options.txt` 决定起点：若只做图像则用 **ImageDecoder**，否则先用 **TextDecoder**。
  - 每次 `next(Generator)` 得到 `DecodePiece(token, next_decoder)`；若 `next_decoder` 非空（例如生成了 BOI），则把当前已生成的 token 拼回 `input_ids`，并切换到 **ImageDecoder** 或回到 **TextDecoder**，继续自回归。

### 4.5 TextDecoder（chameleon/inference/chameleon.py）

- **作用**：从当前 `input_ids`（含完整 prompt）开始，用 **ChameleonGenerator** 自回归生成，直到命中停止条件。
- **ChameleonGenerator 的构造**：
  - `model`：`ChameleonModelAdapter(Transformer, max_seq_len)`；
  - `input_ids`：已对齐的 prompt（可能多 batch）；
  - **stopping_criteria**：`MaxLengthCriteria(max_seq_len)` + 每个 EOS 的 `StopOnEOSAfterBatchIndex(eos_id, [max_prompt_len, ...])`（在 prompt 长度之后若出现 EOS 则停）；
  - **alignment**：`AlignPromptRight(vocab.pad_id)`，即左 pad、右对齐，生成时从「最后一个有效 token」开始算；
  - **token_selector**：`options.txt.greedy` 为 True 则 Argmax，否则 Multinomial（采样）。
- **_logits_processors（文本阶段）**：
  1. **AllowOnlyTokensLogitsProcessor(allowed_tokens)**  
     - 只允许：EOS、文本 token（若 options.txt）、BOI（若 options.img）。  
     - 其余词表位置 logits 置为 **-inf**，相当于**词表维度的剪枝**。
  2. **DisallowTokensAtOrAfterIndexLogitsProcessor([BOI], max_seq_len - 1026)**  
     - 在「序列长度 ≥ max_seq_len - 1026」之后，禁止再生成 BOI，避免再插一张图导致超长。  
     - 这也是**按位置的剪枝**（只针对 BOI）。
  3. 若为 Text 选项：RepetitionPenalty、Temperature、TopP。

- **advance(gen, max_prompt_len)**：在真正开始「生成」之前，先把 Generator 往前推进 `max_prompt_len` 步，即把 prompt 的 token 逐个 emit 出去（不跑模型，只从 cache/输入里取）；之后才开始对「下一个 token」跑 forward 并采样。

因此，在 **TextDecoder** 阶段：

- 允许的 token 集合被剪枝为：**EOS + text_tokens +（可选）BOI**；  
- 在接近最大长度时，BOI 再被按位置禁止。

### 4.6 ImageDecoder（chameleon/inference/chameleon.py）

- **触发**：TextDecoder 某一步采到的 token 是 **BOI**，则下一阶段切到 ImageDecoder。
- **输入**：当前 `input_ids` 已在末尾加上 BOI（若没有会补上）；且会为 **InBatchInstruct CFG** 拆成 3 份：full condition、image-only condition、uncondition，即 3×batch 的序列一起前向，再在 logits 上做 CFG 融合。
- **ChameleonGenerator**：
  - `max_seq_len = max_prompt_len + 1024`（只给 1024 个 image token 的空间）；
  - **logits_processors**：InBatchInstructCFGLogitsProcessor、**AllowOnlyTokensLogitsProcessor(vocab.image_tokens)**（只允许 8192 个 image token，相当于**整段只对 image token 做采样，其他词表位置全部剪枝**）、Temperature、TopP；
  - **token_selector**：ReplicatedInputTokenSelector(Argmax/Multinomial, n=3)，即从 3 路里取主路选的 token，复制 3 份写回。
- **长度控制**：内部 `gen_count` 到 **1024** 时，不再调用 generator，而是**直接构造 EOI token** 并返回，然后切换回 TextDecoder。  
  因此：**每张图严格 1024 个 image token + 1 个 EOI**，没有长度采样。

总结 ImageDecoder 的剪枝：

- 只允许 **image_tokens**（8192 个），其余词表全部 -inf；
- 长度上固定 1024，不生成 EOS/BOI，由代码强制在 1024 步后写 EOI。

### 4.7 ChameleonGenerator 与 model.forward（chameleon/inference/generation.py + model_adapter.py）

- **ChameleonGenerator**（generation.py）：
  - `initialize(input_ids)`：调用 `model.initialize(input_ids)`（在 adapter 里分配 KV cache），并用 **alignment** 做 `prepare_inputs`（右对齐则左 pad），得到 `_inputs`；然后 `_inputs = _inputs[:, :_start_idx]`，这里 **AlignPromptRight** 时 `_start_idx = max(len(sublist))`，即 prompt 长度，因此**保留的是整段 prompt，没有在长度上剪掉**；只是前面是 pad，不参与第一次 forward 的「有效长度」。
  - 每一步：若 `_idx < _start_idx`，直接 emit `_inputs[:, idx]`（不跑模型）；否则对当前 `_inputs` 做一次 **model(inputs)**，取最后一位 logits → logits_processors → softmax → probability_processors → token_selector 得到 next_tokens，拼到 `_inputs` 上，再 alignment.postprocess_inputs，然后 emit 最后一个 token。

- **ChameleonModelAdapter**（model_adapter.py）：
  - **initialize**：按 `batch_size * max_seq_len` 分配 KV cache（`transformer.make_cache`），并设置 `_first_pass = True`。
  - **第一次 __call__**：用 `AttnBias.from_seqlens` 按当前有效长度（prompt 各样本长度）做 causal + padding；只对「有效位置」做一次大 forward，结果写进 cache，并填到 `_local_outputs`；然后把 `q_seqinfo` 改成「每步只问 1 个 query」，供后续步用。
  - **后续 __call__**：只传「当前步的 token」和已有 cache，做 1 token 的 forward，并更新 cache（`k_seqinfo.seqlen.add_(1)`）。  
  这里没有对「历史 token」做剪枝，只是**增量解码 + KV cache**；剪枝发生在 **logits** 上（见上两小节）。

### 4.8 模型结构（chameleon/inference/loader.py + transformer.py）

- **loader.load_model(path)**：读 `params.json`、`consolidate_params.json`，拼成 ModelArgs，再加载 `consolidated.pth` 到 **Transformer**。
- **Transformer**：Embedding → N×TransformerBlock（Attention + FFN）→ Norm → Linear → logits。Attention 使用 xformers 的 RoPE + causal mask，支持 KV cache；**make_cache** 按 `(1, length, n_kv_heads, head_dim)` 每层分配 K/V cache。

## 五、inference.py 中的后处理：split + decode

### 5.1 收集 stream 的 token

- inference.py 对 `model.stream(batch_prompt_ui, options)` 做 for 循环，把每次 yield 的 `token.id` 放进 list，最后 `torch.stack(tokens_list).T` 得到 `tokens`，形状为 `(1, num_generated_tokens)`（batch=1）。

注意：这里**只包含「生成」出来的 token**，不包含 prompt；prompt 在 Chameleon 内部已被 advance 消耗掉，不会从 stream 里再吐出来。

### 5.2 split_token_sequence(tokens, boi, eoi)

- **作用**：把生成序列按 **BOI/EOI** 切成多段，并标记是「文本段」还是「图像段」。
- **规则**：
  - 遇到 **BOI**：若当前有未保存的 text segment，先 push；然后进入「图像段」；
  - 遇到 **EOI** 且当前在图像段：把当前积累的 token 作为一个 **image_seg**（应为 1024 个）push，然后退出图像段；
  - 其余 token 归入当前段（text 或 image）；
  - 最后若还有未保存的，按 in_image_seg 标记为 image_seg 或 text_seg。
- **返回**：`List[(seg_type, seg_tokens)]`，其中 `seg_tokens` 为 `(1, L)` 的 LongTensor；对 image_seg 要求 `L==1024`。

### 5.3 decode 与输出

- **image_seg**：`model.decode_image(seg_tokens)` → TokenManager 里用 VocabTranslation 把 BPE id 转成 codebook id，再用 **ImageTokenizer.pil_from_img_toks**（VQGAN decode）得到 PIL，保存为 `{save_dir}/{seg_id}.png`，并打印路径。
- **text_seg**：`model.decode_text(seg_tokens)` → 先去掉 EOS 及之后的 token，再用文本 tokenizer decode，打印字符串。

## 六、Token 数量与变化小结

| 阶段 | 位置 | Token 数量 / 变化 |
|------|------|-------------------|
| 输入 prompt | tokens_from_ui 后 | 1 (BOS) + Σ(文本 BPE 长度) + 每图 1026 (BOI+1024+EOI)。例：1 段文本 + 1 张图 → 约 1 + N_text + 1026。 |
| 生成（stream） | TextDecoder 输出 | 从 0 开始递增，直到 EOS 或 BOI；若为 BOI，下一个阶段为 ImageDecoder。 |
| 生成（stream） | ImageDecoder 输出 | 固定 **1024** 个 image token + **1** 个 EOI（由代码强制，不经过采样）。 |
| 生成（stream） | 再次 TextDecoder | 又可变长，直到 EOS 或再次 BOI 或达到 max_gen_len。 |
| inference.py 收集的 tokens | 仅「生成」部分 | 不含 prompt；长度 = 所有 TextDecoder 步数 + 1024×图片数 + EOI 数 + 可能的 EOS。 |
| split 后 | 每个 text_seg | 长度不固定，为一段连续文本 token（可含 EOS）。 |
| split 后 | 每个 image_seg | **严格 1024** 个 token（BOI/EOI 不进入 segment，只做边界）。 |

## 七、「剪枝」发生在哪里、是什么

本项目中「剪枝」主要指**对 logits 的约束**（把不允许的 token 设为 -inf），而不是对序列或 KV 的截断：

1. **TextDecoder**  
   - **AllowOnlyTokensLogitsProcessor**：只保留 EOS、text_tokens、以及（若开启图像）BOI；**其余词表全部 -inf**。  
   - **DisallowTokensAtOrAfterIndexLogitsProcessor**：在位置 ≥ `max_seq_len - 1026` 后，禁止 BOI，避免再生成一整张图导致超长。

2. **ImageDecoder**  
   - **AllowOnlyTokensLogitsProcessor(image_tokens)**：只保留 8192 个 image token；**其他词表（含 EOS、BOI、文本 token）全部 -inf**。  
   - 长度上不采样：满 1024 个 image token 后由代码直接写 EOI，不再从 logits 里选。

3. **KV cache**  
   - 预分配长度为 `batch × max_seq_len`，按步写入，没有「删掉历史 token」的剪枝；序列长度由 **MaxLengthCriteria** 和 **StopOnEOS** 等停止条件控制。

## 八、关键常量汇总

- **BOI**：8197（`<racm3:break>`）  
- **EOI**：8196（`<eoss>`）  
- **每张图 token 数**：1024（image） + 1（BOI） + 1（EOI）= 1026（输入/输出中整段）；解码时只取 1024 个 image token。  
- **image_tokens**：词表中 8192 个以 IMGIMG 开头的 id（对应 VQGAN codebook 0–8191 的 BPE 映射）。  
- **max_seq_len / max_gen_len**：inference.py 中设为 4096；文本阶段禁止在「位置 ≥ 4096−1026」后生成 BOI。

以上即从「输入一个 case」到「得到文本 + 图片结果」的完整流程，以及脚本所调用的接口、算法角色和 token 变化与剪枝的梳理。
